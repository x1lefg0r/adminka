# Тема 4: Программирование на Bash

## 4.1. `$@` vs `$*` в двойных кавычках

```bash
args_at "foo bar" "baz"
# "$@" → каждый аргумент отдельно:
# arg: foo bar
# arg: baz

args_star "foo bar" "baz"
# "$*" → все склеены в одну строку:
# arg: foo bar baz
```

- `"$@"` — каждый аргумент отдельная строка, пробелы сохраняются
- `"$*"` — все аргументы склеены через первый символ `$IFS` (обычно пробел)
- Без кавычек — одинаковы, оба делают word splitting

Функция вызывается **один раз** — цикл `for` внутри итерируется по элементам.

**Правило:** всегда используй `"$@"` когда передаёшь аргументы дальше.

## 4.2. Значение по умолчанию через расширение параметров

```bash
${VAR:-default}   # если VAR не задана ИЛИ пустая → использовать "default"
${VAR-default}    # если VAR не задана (но не если пустая) → использовать "default"
${VAR:=default}   # если VAR не задана ИЛИ пустая → присвоить "default" и использовать
${VAR:?error}     # если VAR не задана ИЛИ пустая → вывести ошибку и выйти
${VAR:+other}     # если VAR задана и не пустая → использовать "other" вместо неё
```

Логика:
```
${  VAR  :-  default  }
    ↑    ↑   ↑
    имя  |   значение если условие выполнено
         модификатор (- = читать, = = присвоить, ? = ошибка, + = инверсия)
```

`:` добавляет проверку на пустоту поверх проверки на существование:
```bash
VAR=""
echo ${VAR:-default}   # → default (пустая → сработало)
echo ${VAR-default}    # → ""      (задана, пусть пустая → не сработало)
```

На практике:
```bash
NAME=${1:-"anonymous"}   # первый аргумент или дефолт
DIR=${DIR:-"/tmp"}       # переменная или дефолт

# добавить суффикс только если переменная задана:
DEBUG=1
echo "running${DEBUG:+ in debug mode}"   # → running in debug mode
```

## 4.3. Безопасный перебор файлов с пробелами

```bash
find /path -type f -print0 | while IFS= read -r -d '' file; do
    echo "обрабатываю: $file"
done
```

- `IFS=` — очищаем разделитель, `read` не режет по пробелам
- `read -r` — не интерпретировать `\` как экранирование
- `-d ''` — читать до нулевого байта (`\0`)
- `-print0` — find разделяет нулевым байтом

### IFS — Internal Field Separator:
```bash
IFS=: read a b c <<< "one:two:three"
echo $a   # → one

IFS= read a <<< "one two three"
echo $a   # → one two three (всё в одну переменную, не режет)
```

Простой вариант если пробелов нет:
```bash
for file in /path/*.txt; do
    echo "$file"
done
```

## 4.4. Ассоциативный массив

```bash
declare -A colors
colors["apple"]="red"
colors["banana"]="yellow"

echo ${colors["apple"]}   # → red
```

Это словарь/хэшмап как в JS: `{ apple: "red", banana: "yellow" }`.

Инициализация сразу:
```bash
declare -A colors=([apple]="red" [banana]="yellow")
```

Операции:
```bash
echo ${!colors[@]}   # все ключи (! = инвертировать — дать ключи вместо значений)
echo ${colors[@]}    # все значения
echo ${#colors[@]}   # количество элементов (# = количество)

for key in "${!colors[@]}"; do
    echo "$key → ${colors[$key]}"
done

unset colors["apple"]   # удалить элемент
```

### `declare` флаги:
```bash
declare -A   # Associative array
declare -a   # обычный индексный array
declare -i   # integer
declare -r   # readonly
declare -x   # export
declare -f   # показать функции
```

## 4.5. `local` внутри функции

```bash
NAME="god"

greet() {
    local NAME="anonymous"   # локальная — не трогает глобальную
    echo "hello $NAME"
}

greet       # → hello anonymous
echo $NAME  # → god, не тронуто
```

Без `local` — переменная глобальная, загрязняет пространство имён и перезаписывает внешние переменные.

**Правило:** всегда используй `local` внутри функций для всех переменных.

## 4.6. Целочисленная арифметика

```bash
result=$(( 2 + 2 ))
echo $result   # → 4
```

Операторы:
```bash
$(( a + b ))   # сложение
$(( a - b ))   # вычитание
$(( a * b ))   # умножение
$(( a / b ))   # целочисленное деление
$(( a % b ))   # остаток
$(( a ** b ))  # степень
```

Внутри `(( ))` переменные без `$`:
```bash
echo $(( a + b ))    # правильно
echo $(( $a + $b ))  # работает но избыточно
```

Инкремент и присвоение:
```bash
(( a++ ))
(( a += 5 ))
(( result = a + b ))   # присвоение без $()
echo $result           # читать — с $
```

Логика: пишешь в переменную — без `$`, читаешь — с `$`. Везде в bash одинаково.

## 4.7. Опции оболочки `-e` и `-u`

```bash
#!/bin/bash
set -euo pipefail   # стандартное начало любого серьёзного скрипта
```

- `set -e` (`-o errexit`) — выйти если любая команда вернула ненулевой код
- `set -u` (`-o nounset`) — необъявленная переменная = ошибка (без этого просто пустая строка — молчаливый баг)
- `set -o pipefail` — код возврата конвейера = код последней упавшей команды

```bash
# без pipefail:
cat /несуществующий | grep "foo"   # grep вернул 0 → set -e не сработает
# с pipefail:
cat /несуществующий | grep "foo"   # → ошибка, скрипт остановится
```

Включить/выключить:
```bash
set -e    # включить
set +e    # выключить
```

Посмотреть все опции:
```bash
set -o    # все опции и их состояние
```

## 4.8. Проверка строки на регулярное выражение

```bash
if [[ $string =~ ^[a-z]+[0-9]+$ ]]; then
    echo "совпадение"
fi
```

`=~` — только внутри `[[ ]]`.

Захват групп:
```bash
if [[ "2026-06-25" =~ ^([0-9]{4})-([0-9]{2})-([0-9]{2})$ ]]; then
    echo ${BASH_REMATCH[0]}   # вся строка
    echo ${BASH_REMATCH[1]}   # → 2026
    echo ${BASH_REMATCH[2]}   # → 06
    echo ${BASH_REMATCH[3]}   # → 25
fi
```

Квантификаторы:
```
{4}    # ровно 4 раза
{2}    # ровно 2 раза
{1,3}  # от 1 до 3 раз
+      # одно или более
*      # ноль или более
?      # ноль или одно
^      # начало строки
$      # конец строки
```

## 4.9. `[ ]` vs `[[ ]]`

```bash
[ ]    # старый POSIX, внешняя команда /usr/bin/[
[[ ]]  # bash builtin, расширенный
```

Преимущества `[[ ]]`:

```bash
# 1. Не нужны кавычки (word splitting отключён):
VAR="hello world"
[ $VAR = "hello" ]    # ошибка — word splitting даёт два токена
[[ $VAR = "hello" ]]  # работает

# 2. Логические операторы внутри:
[ $a = "foo" ] && [ $b = "bar" ]
[[ $a = "foo" && $b = "bar" ]]

# 3. Regex:
[[ $str =~ ^[0-9]+$ ]]

# 4. Glob паттерны:
[[ $str == *.txt ]]

# 5. Безопасен с пустыми переменными:
[ -f $FILE ]    # если FILE пустая → синтаксическая ошибка
[[ -f $FILE ]]  # безопасно
```

**Правило:** всегда `[[ ]]` в bash. `[ ]` только для переносимости на sh/dash.

## 4.10. Режим трассировки

```bash
set -x   # включить
set +x   # выключить
```

Показывает каждую команду **после раскрытия переменных** с префиксом `+`:

```bash
set -x
NAME="god"
echo $NAME
# + NAME=god
# + echo god
# god
```

Запустить скрипт с трассировкой не меняя его:
```bash
bash -x ./script.sh
```

Включить только для части скрипта:
```bash
# обычный код
set -x
# трассируемый код
set +x
# обычный код
```

Аналог автоматического `console.log` на каждой строке. Незаменимо когда скрипт работает локально но падает на сервере.
