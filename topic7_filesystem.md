# Тема 7: Файловая система Linux и FHS

## 7.1. Назначения директорий по FHS

**FHS** — Filesystem Hierarchy Standard, стандарт что где должно лежать.

```
/etc     — et cetera — конфигурационные файлы (nginx.conf, sshd_config)
/var     — variable — изменяемые данные (логи, БД, кэш, pid файлы)
/usr     — unix system resources (не user!) — программы и библиотеки
/tmp     — temporary — временные файлы, очищается при перезагрузке
/opt     — optional — стороннее ПО установленное вручную (не через пакетный менеджер)
```

Остальные директории:
```
/bin     — binaries — бинарники
/lib     — libraries — библиотеки
/home    — домашние директории пользователей
/root    — домашняя директория root
/boot    — файлы загрузки (ядро, initramfs, grub)
/dev     — devices — файлы устройств
/proc    — виртуальная ФС процессов
/sys     — виртуальная ФС устройств
/mnt     — mount — точки монтирования
/media   — съёмные носители (USB, CD)
/srv     — service — данные сервисов
/run     — данные запущенных процессов (современная замена /var/run)
/usr/bin     — бинарники пользовательских программ
/usr/lib     — библиотеки
/usr/local   — программы установленные локально (вне пакетного менеджера)
/var/log     — логи
/var/run     — pid файлы
/var/cache   — кэш
```

## 7.2. Использование дискового пространства без пересечения границ ФС

```bash
du -sh /var --one-file-system
```

- `-s` — summary, только итог для директории
- `-h` — human readable
- `--one-file-system` — не заходить в примонтированные подразделы

Без флага `du` посчитает все вложенные примонтированные разделы тоже.

## 7.3. Структура записи в /etc/fstab

fstab — текстовый файл, список "что монтировать при загрузке". systemd читает его при старте.

```
/dev/sda1   /       ext4    defaults    0    1
↑           ↑       ↑       ↑           ↑    ↑
устройство  точка   тип ФС  опции      dump pass
            монт.
```

### Поля:
- **устройство** — `/dev/sda1`, `UUID=...`, `LABEL=...`
- **точка монтирования** — куда монтировать
- **тип ФС** — `ext4`, `xfs`, `tmpfs`, `swap`, `nfs`
- **опции** — `defaults`, `noatime`, `ro`, `noexec`
- **dump** — устаревшее, всегда `0`
- **pass** — порядок проверки `fsck`:
  ```
  0 — не проверять (swap, tmpfs, nfs)
  1 — первым (только корневая /)
  2 — после корневой (остальные разделы)
  ```

### Лучше использовать UUID:
```bash
blkid   # показать UUID всех устройств
```

UUID не меняется при добавлении нового диска — `/dev/sda` может измениться.

### Опции монтирования:
```
defaults  — стандартный набор (rw,suid,exec,auto,nouser,async)
ro        — read only
rw        — read write
noatime   — не обновлять atime
nodiratime— не обновлять atime для директорий
noexec    — нельзя запускать бинарники
nosuid    — игнорировать SUID биты
nodev     — игнорировать файлы устройств
sync      — синхронная запись (медленно но надёжно)
async     — асинхронная запись (дефолт)
auto      — монтировать при mount -a
noauto    — не монтировать при mount -a
user      — обычный пользователь может монтировать
```

## 7.4. Журналирование ext4 vs XFS

Журналирование — защита целостности ФС при внезапном отключении. Пишет в журнал что собирается сделать, при сбое восстанавливается из журнала.

### ext4 — три режима:
```
data=writeback  — только метаданные, данные могут быть старые после сбоя
data=ordered    — дефолт, данные пишутся до метаданных (без журнала)
data=journal    — всё журналируется, максимальная надёжность, медленнее
```

`data=journal` надёжнее потому что журналирует сами байты файла, не только метаданные. При `data=ordered` блоки данных могут быть записаны наполовину.

### XFS:
- Журналирует только метаданные (архитектурное решение, не ограничение)
- Лучше на параллельной записи — несколько потоков одновременно
- Лучше на больших файлах
- **Нельзя уменьшить** — только увеличить (`xfs_growfs`)
- Быстрее восстанавливается после сбоя
- Inode создаются динамически — нельзя исчерпать

### ext4 vs XFS — inode:
```bash
# ext4 — фиксированное количество inode при форматировании:
df -i   # можно исчерпать inode даже если место есть
touch newfile   # No space left on device — хотя df -h показывает место

# XFS — динамические inode, исчерпать практически невозможно
```

### Когда что:
```
ext4  — общего назначения, стабильная, можно уменьшить раздел
XFS   — большие файлы, высокая нагрузка записи, параллельность
```

Ubuntu/Debian — ext4 по умолчанию. RHEL/CentOS — XFS по умолчанию.

## 7.5. Перемонтировать корневую ФС в режим чтения-записи

```bash
mount -o remount,rw /
```

- `-o` — options, передать опции монтирования
- `remount` — перемонтировать с новыми опциями не размонтируя

Корневую ФС нельзя размонтировать пока система работает — `remount` единственный способ изменить опции на лету.

```bash
mount | grep "on / "   # проверить текущие опции
# → /dev/sda1 on / type ext4 (ro,relatime)
mount -o remount,rw /
# → (rw,relatime)
```

### Флаги mount:
```bash
-t ext4           # type — тип ФС
-o options        # options — опции
-a                # all — примонтировать всё из /etc/fstab
-r                # read only
-w                # read write
-l                # list — показать примонтированные ФС
```

## 7.6. Bind-монтирование

```bash
mount --bind /opt/nginx-1.25 /opt/nginx
```

Два пути к одним данным — оригинал никуда не девается:
```
/opt/nginx-1.25  ─┐
                   ├──→ одни и те же данные на диске
/opt/nginx       ─┘
```

Убрать bind-mount:
```bash
umount /opt/nginx   # оригинал /opt/nginx-1.25 жив
```

### Bind-mount vs симлинк:
```
симлинк   — файл с текстом пути, разыменовывается на уровне VFS
bind-mount — ядро регистрирует точку монтирования, прозрачно для всего
```

Симлинк не работает внутри chroot/контейнера — абсолютный путь цели не существует внутри изолированного окружения. Bind-mount работает — ядро уже знает куда это указывает до chroot.

### Подобие жёсткой ссылки для директорий:
```
жёсткая ссылка  — два имени одного inode (для файлов)
bind-mount       — два пути к одной директории на уровне ядра
```

### chroot:
```bash
chroot /jail /bin/bash   # / для этого bash = /jail
exit                      # вернуться обратно
```

Процесс не может выйти за пределы своего "корня". Внутри должны быть все нужные бинарники и библиотеки.

## 7.7. Монтирование раздела

```bash
mkdir -p /mnt/data
mount -t ext4 -o noatime /dev/sdb1 /mnt/data
```

Точка монтирования — существующая директория куда "прикрепляется" ФС. До монтирования — пустая директория. После — показывает содержимое диска.

```bash
mount | grep sdb1    # проверить
umount /mnt/data     # размонтировать
```

### Как ядро понимает что это другая ФС:
Читает суперблок — первый блок ФС, содержит тип, размер, где inode таблица, где журнал. По нему ядро определяет отдельную ФС.

```bash
ls -la /dev/sdb1
# brw-rw---- ... /dev/sdb1
# b — block device (первая буква в правах)
```

## 7.8. tmpfs

ФС которая живёт в RAM:

```bash
mount -t tmpfs -o size=512m tmpfs /tmp
```

Характеристики:
- Молниеносно быстрая — всё в памяти
- Исчезает при перезагрузке
- Размер динамический
- Может использовать swap если RAM заканчивается

По умолчанию:
```bash
df -h
# tmpfs  /tmp      # временные файлы
# tmpfs  /run      # pid файлы, сокеты
# tmpfs  /dev/shm  # разделяемая память между процессами
```

```
tmpfs   — быстро, не персистентно, занимает RAM
ext4    — медленнее, персистентно, занимает диск
```

## 7.10. Swap-пространство

Когда RAM заканчивается — ядро выгружает давно неиспользуемые страницы памяти в swap. При обращении к ним — читает обратно в RAM (page fault → swap in). Процесс не знает что его память была в swap.

**Thrashing** — система непрерывно гоняет страницы между RAM и диском:
```bash
vmstat 1
# si — swap in (из swap в RAM)
# so — swap out (из RAM в swap)
# высокие si/so = thrashing
```

### Создать swap-файл 1GB:
```bash
# быстрый способ:
fallocate -l 1G /swapfile

# надёжный способ (dd пишет нули побайтово):
dd if=/dev/zero of=/swapfile bs=1M count=1024
# if = input file (/dev/zero — генерирует нули)
# of = output file
# bs = block size
# count = количество блоков (1024 × 1MB = 1GB)

chmod 600 /swapfile   # только root
mkswap /swapfile      # пометить как swap
swapon /swapfile      # включить

swapon --show         # проверить
free -h               # посмотреть память и swap
```

Постоянно — в `/etc/fstab`:
```
/swapfile  none  swap  sw  0  0
```

### /dev/zero vs /dev/null:
```
/dev/null  — поглощает всё (чёрная дыра)
/dev/zero  — генерирует нули (бесконечный источник)
```
